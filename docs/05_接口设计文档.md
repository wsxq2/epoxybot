## 1. ROS2接口定义

### 1.1 自定义消息类型

#### 1.1.1 DefectArray.msg

用于缺陷检测结果发布。

```msg
# DefectArray.msg
std_msgs/Header header

# 图像元信息
uint32 image_width      # 原始图像宽度
uint32 image_height     # 原始图像高度
string camera_frame_id  # 相机坐标系

# 缺陷列表
Defect[] defects

---

# Defect.msg（嵌套消息）
uint32 id               # 缺陷唯一ID
string class_name       # 类别名称（crack/hole/unevenness/discoloration）
uint8 class_id          # 类别ID（0-3）
float32 confidence      # 置信度（0.0-1.0）

# 边界框（像素坐标）
uint32 bbox_x           # 左上角X坐标
uint32 bbox_y           # 左上角Y坐标
uint32 bbox_width       # 宽度
uint32 bbox_height      # 高度

# 真实世界坐标（可选，需要相机标定）
geometry_msgs/Point position_3d  # 3D坐标（相机坐标系）

# 缺陷属性
float32 area            # 面积（平方毫米）
float32 severity        # 严重程度（0.0-1.0）
```

**使用示例：**

```python
from epoxybot_interfaces.msg import DefectArray, Defect

defect_array = DefectArray()
defect_array.header.stamp = self.get_clock().now().to_msg()
defect_array.header.frame_id = "camera_link"
defect_array.image_width = 8688
defect_array.image_height = 7680

defect = Defect()
defect.id = 1
defect.class_name = "crack"
defect.class_id = 0
defect.confidence = 0.95
defect.bbox_x = 1024
defect.bbox_y = 2048
defect.bbox_width = 256
defect.bbox_height = 128
defect.area = 32.5  # mm²
defect.severity = 0.8

defect_array.defects.append(defect)
```

#### 1.1.2 EmergencyStopAll.msg

系统级紧急停止消息。

```msg
# EmergencyStopAll.msg
std_msgs/Header header

# 停止原因
string reason           # 停止原因描述
uint8 stop_level        # 停止级别（0=软停止, 1=硬停止）

# 停止范围
string[] target_robots  # 目标机器人命名空间（空数组表示全部）

# 恢复方式
bool auto_recovery      # 是否允许自动恢复
float32 recovery_delay  # 自动恢复延迟（秒）
```

**使用示例：**

```python
from epoxybot_interfaces.msg import EmergencyStopAll

stop_msg = EmergencyStopAll()
stop_msg.header.stamp = self.get_clock().now().to_msg()
stop_msg.reason = "Obstacle detected in safety zone"
stop_msg.stop_level = 1  # 硬停止
stop_msg.target_robots = []  # 停止所有机器人
stop_msg.auto_recovery = False
```

### 1.2 自定义服务类型

#### 1.2.1 StartTask.srv

启动作业任务服务。

```srv
# 请求
string task_name              # 任务名称
uint8 task_type               # 任务类型（0=喷涂, 1=刮涂, 2=打磨）
geometry_msgs/Pose[] waypoints  # 路径点列表
float32 work_speed            # 作业速度（m/s）
float32 spray_rate            # 喷涂速率（ml/s，仅喷涂任务）

# 任务参数
string parameter_json         # JSON格式的任务参数

---
# 响应
bool success                  # 是否成功
string task_id                # 任务ID（成功时返回）
string message                # 状态消息
```

**使用示例：**

```python
from epoxybot_interfaces.srv import StartTask
from geometry_msgs.msg import Pose

# 客户端调用
request = StartTask.Request()
request.task_name = "Wall Section A1"
request.task_type = 0  # 喷涂
request.work_speed = 0.2

pose1 = Pose()
pose1.position.x = 1.0
pose1.position.y = 2.0
request.waypoints.append(pose1)

future = client.call_async(request)
```

#### 1.3.1 NavigateToWaypoint.action

导航到指定路径点动作。

```action
# 目标
geometry_msgs/PoseStamped target_pose  # 目标位置
float32 max_speed                      # 最大速度（m/s）
bool avoid_obstacles                   # 是否避障

---
# 结果
bool success                           # 是否成功
float32 total_distance                 # 总距离（m）
builtin_interfaces/Duration total_time # 总用时
string message                         # 结果消息

---
# 反馈
geometry_msgs/PoseStamped current_pose # 当前位置
float32 distance_remaining             # 剩余距离（m）
builtin_interfaces/Duration time_remaining  # 预计剩余时间
float32 current_speed                  # 当前速度（m/s）
```

**使用示例：**

```python
from epoxybot_interfaces.action import NavigateToWaypoint
from rclpy.action import ActionClient

# 创建动作客户端
action_client = ActionClient(node, NavigateToWaypoint, 'navigate_to_waypoint')

# 发送目标
goal_msg = NavigateToWaypoint.Goal()
goal_msg.target_pose.header.frame_id = "map"
goal_msg.target_pose.pose.position.x = 5.0
goal_msg.target_pose.pose.position.y = 3.0
goal_msg.max_speed = 0.5
goal_msg.avoid_obstacles = True

future = action_client.send_goal_async(goal_msg, feedback_callback=feedback_cb)
```

## 2. Modbus协议规范

### 2.1 协议参数

| 参数 | 值 | 说明 |
|------|-----|-----|
| 协议类型 | Modbus TCP | 基于TCP/IP的Modbus协议 |
| 端口 | 502 | 标准Modbus端口 |
| 从站ID | 1 | PLC从站地址 |
| 超时时间 | 500ms | 单次请求超时 |
| 重试次数 | 3 | 失败后重试次数 |

---

### 2.2 功能码

本系统使用以下Modbus功能码：

| 功能码 | 名称 | 用途 |
|--------|------|------|
| 0x03 | Read Holding Registers | 读取PLC寄存器（位置、速度、传感器） |
| 0x06 | Write Single Register | 写入单个寄存器 |
| 0x10 | Write Multiple Registers | 批量写入寄存器（提高效率） |

---

### 2.3 寄存器地址映射

#### 2.3.1 侧墙/地坪平台寄存器表

**关节状态反馈（0x1000 - 0x1FFF）：**

| 地址 | 数据类型 | 名称 | 单位 | 说明 |
|------|---------|------|------|------|
| 0x1000 | int16 | 左前转向位置 | 脉冲 | 转换系数：÷10000 → rad |
| 0x1001 | int16 | 左前转向速度 | 脉冲/秒 | 转换系数：÷1000 → rad/s |
| 0x1002 | int16 | 右前转向位置 | 脉冲 | 转换系数：÷10000 → rad |
| 0x1003 | int16 | 右前转向速度 | 脉冲/秒 | 转换系数：÷1000 → rad/s |
| 0x1004 | int16 | 左前驱动位置 | 脉冲 | 转换系数：÷10000 → m |
| 0x1005 | int16 | 左前驱动速度 | 脉冲/秒 | 转换系数：÷1000 → m/s |
| ... | ... | ... | ... | 每个关节占2个寄存器 |
| 0x1010-0x1015 | int16 | 机械臂关节1-3 | 脉冲 | 位置+速度 |

**关节命令写入（0x2000 - 0x2FFF）：**

| 地址 | 数据类型 | 名称 | 单位 | 说明 |
|------|---------|------|------|------|
| 0x2000 | int16 | 左前转向位置命令 | 脉冲 | rad × 10000 → 脉冲 |
| 0x2001 | int16 | 左前转向速度命令 | 脉冲/秒 | rad/s × 1000 → 脉冲/秒 |
| ... | ... | ... | ... | 与状态反馈对应 |

**传感器数据（0x3000 - 0x3FFF）：**

| 地址 | 数据类型 | 名称 | 单位 | 说明 |
|------|---------|------|------|------|
| 0x3000-0x3009 | uint16 | 激光测距仪1-10 | mm | 直接读取，无需转换 |
| 0x3010 | uint16 | 喷涂压力传感器 | 0.01N | 寄存器值 ÷ 100 → N |
| 0x3011 | uint16 | 喷涂流量传感器 | 0.1ml/s | 寄存器值 ÷ 10 → ml/s |

**GPIO（0x4000 - 0x4FFF 读取，0x5000 - 0x5FFF 写入）：**

| 地址 | 位 | 名称 | 类型 | 说明 |
|------|-----|------|-----|------|
| 0x4000 | Bit 0 | 急停按钮 | 输入 | 1=按下 |
| 0x4000 | Bit 1 | X轴正限位 | 输入 | 1=触发 |
| 0x4000 | Bit 2 | X轴负限位 | 输入 | 1=触发 |
| 0x4000 | Bit 3 | Y轴正限位 | 输入 | 1=触发 |
| 0x4000 | Bit 4 | Y轴负限位 | 输入 | 1=触发 |
| 0x5000 | Bit 0 | 喷涂阀门 | 输出 | 1=开启 |
| 0x5000 | Bit 1 | 警示灯 | 输出 | 1=开启 |
| 0x5000 | Bit 2 | 蜂鸣器 | 输出 | 1=开启 |

**电池状态（0x6000 - 0x6FFF）：**

| 地址 | 数据类型 | 名称 | 单位 | 说明 |
|------|---------|------|------|------|
| 0x6000 | uint16 | 电池电压 | 0.1V | 寄存器值 ÷ 10 → V |
| 0x6001 | int16 | 电池电流 | 0.1A | 寄存器值 ÷ 10 → A（负值=放电） |
| 0x6002 | uint16 | 电池SOC | 0.1% | 寄存器值 ÷ 10 → % |
| 0x6003 | uint16 | 电池温度 | 0.1°C | 寄存器值 ÷ 10 → °C |
| 0x6004 | uint16 | 电池状态 | 位域 | Bit0=充电中, Bit1=市电连接 |

### 2.4 Modbus请求示例

#### 2.4.1 读取关节位置（功能码0x03）

**请求帧（读取地址0x1000，数量10）：**

```
| Transaction ID | Protocol ID | Length | Unit ID | Function | Start Addr | Quantity |
|----------------|-------------|--------|---------|----------|------------|----------|
|     0x0001     |    0x0000   |  0x06  |  0x01   |   0x03   |   0x1000   |  0x000A  |
```

**响应帧：**

```
| Transaction ID | Protocol ID | Length | Unit ID | Function | Byte Count | Data[0..19] |
|----------------|-------------|--------|---------|----------|------------|-------------|
|     0x0001     |    0x0000   |  0x17  |  0x01   |   0x03   |    0x14    | 20字节数据   |
```

---

#### 2.4.2 写入速度命令（功能码0x10）

**请求帧（写入地址0x2001，数量6，值=[1000, 1000, 1000, 1000, 1000, 1000]）：**

```
| Transaction ID | Protocol ID | Length | Unit ID | Function | Start Addr | Quantity | Byte Count | Data      |
|----------------|-------------|--------|---------|----------|------------|----------|------------|-----------|
|     0x0002     |    0x0000   |  0x13  |  0x01   |   0x10   |   0x2001   |  0x0006  |    0x0C    | 12字节值  |
```

---

### 2.5 错误处理

**异常响应码：**

| 异常码 | 名称 | 说明 | 处理方式 |
|--------|------|------|----------|
| 0x01 | Illegal Function | 不支持的功能码 | 检查功能码配置 |
| 0x02 | Illegal Data Address | 无效的寄存器地址 | 检查地址映射表 |
| 0x03 | Illegal Data Value | 无效的数据值 | 检查数据范围 |
| 0x04 | Slave Device Failure | PLC故障 | 重启PLC或切换安全模式 |
| 0x0B | Gateway Target Failed | 网络故障 | 重连3次，失败后告警 |

## 3. 管理平台HTTP API

### 4.2 PostgreSQL表结构

#### 4.2.1 robots表

```sql
CREATE TABLE robots (
    id VARCHAR(50) PRIMARY KEY,
    type VARCHAR(20) NOT NULL,  -- sidewall/floor/epoxy
    namespace VARCHAR(50) NOT NULL,
    plc_ip VARCHAR(15) NOT NULL,
    status VARCHAR(20) DEFAULT 'offline',  -- online/offline/error
    battery_percentage FLOAT,
    position_x FLOAT,
    position_y FLOAT,
    position_theta FLOAT,
    current_task_id VARCHAR(50),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_robots_status ON robots(status);
```

所有接口遵循语义化版本规范（Semantic Versioning）：

- **主版本号**：不兼容的API修改
- **次版本号**：向下兼容的功能新增
- **修订号**：向下兼容的问题修正
