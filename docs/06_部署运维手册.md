### 1.2 软件安装

#### 1.2.1 工控机软件安装

**操作系统安装**

```bash
# 1. 下载Ubuntu 22.04 LTS镜像
wget https://releases.ubuntu.com/22.04/ubuntu-22.04.3-desktop-amd64.iso

# 2. 制作启动U盘
sudo dd if=ubuntu-22.04.3-desktop-amd64.iso of=/dev/sdX bs=4M status=progress

# 3. 安装系统（按提示操作）
# - 分区: / (100GB), /home (剩余空间), swap (16GB)
# - 用户名: epoxybot
# - 主机名: epoxybot-ipc-{robot_id}
```

**安装ROS2 Humble**

```bash
# 设置locale
sudo apt update && sudo apt install locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8

# 添加ROS2 apt源
sudo apt install software-properties-common
sudo add-apt-repository universe
sudo apt update && sudo apt install curl -y
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
  -o /usr/share/keyrings/ros-archive-keyring.gpg

echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \
  http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" \
  | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

# 安装ROS2 Humble Desktop
sudo apt update
sudo apt install ros-humble-desktop -y

# 安装开发工具
sudo apt install python3-colcon-common-extensions -y
sudo apt install python3-rosdep python3-vcstool -y

# 初始化rosdep
sudo rosdep init
rosdep update

# 设置环境变量
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

**安装依赖库**

```bash
# NAV2导航栈
sudo apt install ros-humble-navigation2 ros-humble-nav2-bringup -y

# 激光雷达驱动
sudo apt install ros-humble-rslidar-sdk -y

# 相机驱动
sudo apt install ros-humble-camera-info-manager -y

# 其他工具
sudo apt install ros-humble-rqt* -y
sudo apt install ros-humble-rviz2 -y
```

**安装YOLO v12**

```bash
# 安装PyTorch
pip3 install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118

# 安装YOLO
pip3 install ultralytics

# 测试安装
python3 -c "from ultralytics import YOLO; print('YOLO安装成功')"
```

**安装FAST-LIVO2（Docker方式）**

```bash
# 安装Docker
sudo apt install docker.io -y
sudo systemctl enable docker
sudo usermod -aG docker $USER

# 拉取FAST-LIVO2镜像
docker pull ros:noetic

# 创建容器
docker run -it --name fast-livo2 \
  --network=host \
  -v /home/epoxybot/fast_livo2:/workspace \
  ros:noetic

# 容器内安装FAST-LIVO2
# (详细步骤见官方文档)
```

**安装胶泥项目软件**

```bash
# 创建工作空间
mkdir -p ~/epoxybot_ws/src
cd ~/epoxybot_ws/src

# 克隆代码（假设使用Git）
git clone https://github.com/your-org/epoxybot.git

# 编译
cd ~/epoxybot_ws
colcon build --symlink-install

# 设置环境变量
echo "source ~/epoxybot_ws/install/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

### 1.3 系统配置

#### 1.3.1 网络配置

**工控机网络配置**

```bash
# 编辑 /etc/netplan/01-netcfg.yaml
sudo nano /etc/netplan/01-netcfg.yaml
```

```yaml
network:
  version: 2
  renderer: NetworkManager
  ethernets:
    eth0:  # PLC连接
      dhcp4: no
      addresses:
        - 192.168.1.10/24
  wifis:
    wlan0:  # WIFI6
      dhcp4: yes
      access-points:
        "Epoxybot_Work":
          password: "your_password"
```

```bash
# 应用配置
sudo netplan apply
```

**DDS配置**

创建 `~/epoxybot_ws/src/epoxybot/config/fastdds.xml`:

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<dds>
    <profiles xmlns="http://www.eprosima.com/XMLSchemas/fastRTPS_Profiles">
        <transport_descriptors>
            <transport_descriptor>
                <transport_id>UDPv4Transport</transport_id>
                <type>UDPv4</type>
            </transport_descriptor>
        </transport_descriptors>
        
        <participant profile_name="epoxybot_profile">
            <rtps>
                <userTransports>
                    <transport_id>UDPv4Transport</transport_id>
                </userTransports>
                <useBuiltinTransports>false</useBuiltinTransports>
            </rtps>
        </participant>
    </profiles>
</dds>
```

设置环境变量:

```bash
echo 'export FASTRTPS_DEFAULT_PROFILES_FILE=~/epoxybot_ws/src/epoxybot/config/fastdds.xml' >> ~/.bashrc
source ~/.bashrc
```

#### 1.3.2 机器人参数配置

编辑 `~/epoxybot_ws/src/epoxybot/config/robot_params.yaml`:

```yaml
robot:
  robot_id: "sidewall"  # 或 "floor", "epoxy"
  
  # 底盘参数
  chassis:
    wheel_radius: 0.15  # m
    wheel_base: 0.8     # m
    max_linear_vel: 1.0  # m/s
    max_angular_vel: 1.0  # rad/s
  
  # PLC配置
  plc:
    ip: "192.168.1.100"
    port: 502
    unit_id: 1
    timeout: 1.0  # s
  
  # 激光雷达配置
  lidar:
    front:
      ip: "192.168.1.201"
      port: 2368
    rear:
      ip: "192.168.1.202"
      port: 2368
  
  # 相机配置
  camera:
    left:
      device_id: "/dev/video0"
      resolution: [6480, 4860]  # 31MP
    right:
      device_id: "/dev/video1"
      resolution: [6480, 4860]
  
  # YOLO配置
  yolo:
    model_path: "~/epoxybot_ws/models/yolo_v12.pt"
    confidence_threshold: 0.5
    device: "cuda:0"  # 或 "cpu"
  
  # NAV2配置
  nav2:
    global_planner: "GridBased"
    local_planner: "DWB"
    max_planning_time: 5.0  # s
```

## 2. 启动流程

### 2.1 系统启动

#### 2.1.1 工控机启动

```bash
# 1. 启动ROS2核心
ros2 daemon start

# 2. 启动激光雷达驱动
ros2 launch epoxybot_bringup lidar.launch.py

# 3. 启动相机驱动
ros2 launch epoxybot_bringup camera.launch.py

# 4. 启动SLAM
ros2 launch epoxybot_slam slam.launch.py

# 5. 启动导航
ros2 launch epoxybot_navigation navigation.launch.py

# 6. 启动控制模块
ros2 launch epoxybot_control control.launch.py

# 7. 启动缺陷检测
ros2 launch epoxybot_defect_detection defect_detection.launch.py
```

**或使用一键启动脚本**:

```bash
# 启动所有模块
ros2 launch epoxybot_bringup epoxybot.launch.py robot_id:=sidewall
```

```bash
# 启动控制界面
ros2 run epoxybot_control control_ui
```

## 4. 故障排查

### 4.1 常见故障

#### 4.1.1 激光雷达无数据

**症状**: `/scan` Topic无数据

**排查步骤**:
1. 检查网络连接
   ```bash
   ping 192.168.1.201
   ```
2. 检查驱动是否启动
   ```bash
   ros2 node list | grep lidar
   ```
3. 查看日志
   ```bash
   ros2 topic echo /rosout | grep lidar
   ```

**解决方案**:
- 重启激光雷达驱动
  ```bash
  ros2 lifecycle set /lidar_driver configure
  ros2 lifecycle set /lidar_driver activate
  ```
- 检查雷达电源和网线

---

#### 4.1.2 相机图像模糊

**症状**: 缺陷检测精度下降

**排查步骤**:
1. 查看原始图像
   ```bash
   ros2 run rqt_image_view rqt_image_view
   ```
2. 检查镜头清洁度
3. 检查相机参数

**解决方案**:
- 清洁镜头
- 重新标定相机
  ```bash
  ros2 run camera_calibration cameracalibrator \
    --size 8x6 --square 0.108 \
    image:=/camera/image_raw
  ```

---

#### 4.1.3 PLC通信超时

**症状**: 控制指令无响应

**排查步骤**:
1. 检查网络连接
   ```bash
   ping 192.168.1.100
   ```
2. 检查Modbus通信
   ```bash
   modpoll -m tcp -a 1 -r 0 -c 1 192.168.1.100
   ```
3. 查看PLC状态指示灯

**解决方案**:
- 重启PLC
- 检查网线和交换机
- 检查Modbus配置参数

---

#### 4.1.4 导航定位丢失

**症状**: 机器人位置跳变或无法导航

**排查步骤**:
1. 检查激光雷达数据
2. 查看定位误差
   ```bash
   ros2 topic echo /amcl_pose
   ```
3. 检查地图是否正确加载

**解决方案**:
- 重新定位
  ```bash
  ros2 service call /reinitialize_global_localization \
    std_srvs/srv/Empty
  ```
- 重新建图（如果环境变化）

---

#### 4.1.5 WIFI连接不稳定

**症状**: 远程监控延迟高或断连

**排查步骤**:
1. 检查信号强度
   ```bash
   iwconfig wlan0
   ```
2. 检查网络延迟
   ```bash
   ping -c 10 192.168.1.1
   ```
3. 检查路由器状态

**解决方案**:
- 调整路由器位置
- 切换到5GHz频段
- 使用Mesh扩展
- 切换到4G/5G备用网络

---

### 4.2 紧急情况处理

#### 4.2.1 紧急停止

**触发方式**:
1. 按下硬件急停按钮
2. 控制系统点击"紧急停止"
3. 远程管理平台点击"远程制动"

**后续处理**:
1. 检查现场情况
2. 排除故障原因
3. 复位急停按钮
4. 在控制系统中点击"复位"
5. 重新启动作业

---

#### 4.2.2 碰撞处理

**处理步骤**:
1. 立即按下急停
2. 检查设备损坏情况
3. 记录碰撞位置和原因
4. 修复损坏部件
5. 调整导航参数或地图
6. 进行测试运行

## 5. 数据管理

### 5.1 数据备份

#### 5.1.1 自动备份脚本

创建 `/home/epoxybot/scripts/backup.sh`:

```bash
#!/bin/bash

BACKUP_DIR=~/epoxybot_ws/backup
DATE=$(date +%Y%m%d_%H%M%S)

# 备份数据库
cp ~/epoxybot_ws/data/epoxybot.db \
   $BACKUP_DIR/db/epoxybot_$DATE.db

# 备份地图
cp -r ~/epoxybot_ws/maps \
   $BACKUP_DIR/maps_$DATE

# 备份配置文件
cp -r ~/epoxybot_ws/src/epoxybot/config \
   $BACKUP_DIR/config_$DATE

# 压缩备份
tar -czf $BACKUP_DIR/backup_$DATE.tar.gz \
   $BACKUP_DIR/db/epoxybot_$DATE.db \
   $BACKUP_DIR/maps_$DATE \
   $BACKUP_DIR/config_$DATE

# 删除临时文件
rm -rf $BACKUP_DIR/db/epoxybot_$DATE.db
rm -rf $BACKUP_DIR/maps_$DATE
rm -rf $BACKUP_DIR/config_$DATE

# 保留最近30天的备份
find $BACKUP_DIR -name "backup_*.tar.gz" -mtime +30 -delete

echo "备份完成: backup_$DATE.tar.gz"
```

#### 5.1.2 设置定时备份

```bash
# 添加到crontab
crontab -e

# 每天凌晨2点备份
0 2 * * * /home/epoxybot/scripts/backup.sh >> /home/epoxybot/backup.log 2>&1
```

### 5.2 数据恢复

```bash

# 恢复数据库

# 恢复地图
```
